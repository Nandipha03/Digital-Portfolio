# Reflections – Digital Portfolio

## 1. Reflection: Creating My CV in Markdown (STAR Method)

**Situation:**  
As part of my digital portfolio, I was required to create a professional CV using Markdown on GitHub.

**Task:**  
I needed to structure my academic, personal, and professional information in a clear and organised format using Markdown syntax.

**Action:**  
I used Markdown elements such as headings, bullet points, and sections to structure my CV into categories including education, experience, skills, and references. I focused on clarity, readability, and professional presentation while learning how Markdown is used in real development environments.

**Result:**  
I successfully created a structured and professional CV. This improved my understanding of Markdown, strengthened my attention to detail, and helped me present information in a more organised and technical format.

## 2. Reflection: Mock Interview Experience (STAR Method)

**Situation:**  
I participated in a mock interview as part of my work readiness preparation.

**Task:**  
The goal was to improve my communication skills, build confidence, and practice responding to common interview questions in a professional setting.

**Action:**  
I prepared by thinking through possible interview questions and structuring my answers clearly. During the interview, I focused on speaking confidently, maintaining professionalism, and expressing my experiences in a structured manner.

**Result:**  
The experience helped me improve my confidence, communication skills, and ability to present myself professionally. It also highlighted areas where I can continue improving my interview technique.

## 3. Reflection: GitHub Pages Deployment (STAR Method)

**Situation:**  
I needed to publish my digital portfolio using GitHub Pages.

**Task:**  
The task was to deploy my repository as a live website to showcase my CV, reflections, and supporting evidence.

**Action:**  
I organised my repository, ensured all files were correctly structured, and used GitHub Pages settings to publish the site from the main branch. I also reviewed how the changes would appear in a live environment.

**Result:**  
I successfully deployed my portfolio using GitHub Pages. This improved my understanding of version control, deployment processes, and how static websites are hosted online.
